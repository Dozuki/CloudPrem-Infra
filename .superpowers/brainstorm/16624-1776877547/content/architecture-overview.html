Key Design Decisions

  • Shared VPC вҖ” both clusters in the same VPC for direct access to shared data layer
  • Separate NLBs вҖ” each cluster gets its own NLB; Route53 weighted routing controls split
  • Shared Aurora вҖ” both clusters point at the same Aurora instance (backward-compatible migrations)
  • Shared ElastiCache вҖ” memcached is a cache, safe to share (cache invalidation is worst case)
  • Shared S3 вҖ” buckets are global, both clusters use the same buckets
  • Green spins up on deploy вҖ” no idle resources when not deploying
  • Green tears down after cutover вҖ” or becomes the new "blue" for next deployment
